## 패턴
- 추상팩토리 패턴
	- 상위 객체에서 객체를 추상화하고 하위 클래스에서 구체적인 구현을 담당하는 것
- MVC
	- 모델 뷰 컨트롤러
		- 모델 : 데이터 처리 영역
			- DAO: Data Access Object 데이터베이스 연동 클래스
			- DO : Data Object 엔티티 클래스, 데이터 구조를 표현
		- 뷰
			- 화면을 구성하는 영역
			- jsp html 등
			- 기본적으로 모델,컨트롤러와 종속성이 없도록 구현.
		- 컨트롤러
			- 사용자 요청의 중심에 위치
			- 사용자 요청이 컨트롤러 -> 뷰로 이동해야함
			- 모델 -> 컨트롤러 -> 뷰
			- 뷰에 전달할대 DO
	- 목적
		- 화면과 데이터 처리를 분리하여 코드간의 종속성을 줄임.
